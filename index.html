About This Debug Router

This page exists purely for testing. Its main purpose is to let you open the customer view and the admin view at the same time in separate tabs, so you can interact with both sides of the system simultaneously without having to log out and log back in between them.

Each button checks your current localStorage session before deciding where to send you. If you are already logged in as a customer it will take you straight to the customer page. If not, it will send you to the login page first. The same logic applies to the admin button.

In a real production setup, the auth page would include this session check so that a user who is already logged in never sees the login screen again. It would look like this:

if (getIsAdminLoggedIn()) { location.replace("../html/admin.html"); } else if (getLoggedInCustomer()) { location.replace("../html/customer.html"); }

This block was intentionally left out during development because having it active makes testing harder. If it were in place, every time a tester visits the auth page while already logged in, they would get redirected away immediately with no way to switch accounts without manually clearing localStorage.

When the system is ready for a real deployment, that block should be added back into auth.js to give users the proper stay-logged-in experience.
